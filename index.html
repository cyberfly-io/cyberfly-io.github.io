CAPABILITIES

Architecture & Platform Capabilities

A comprehensive overview of device, data, automation, edge, security, developer, and ecosystem features.

1. Device Management

  • Multi-protocol Support – MQTT, HTTP/WebSocket for device communication.
  • Dynamic Device Registration – Devices can self-register or be provisioned via the UI.
  • Secure Key Exchange – Public/private key encryption for device-specific integration secrets.
  • Over-the-Air Updates (OTA) – Firmware updates from the cloud.
  • Captive Portal Configuration – On-device Wi-Fi provisioning and re-configuration.

2. Data Handling

  • Hybrid Storage Architecture – Uses Redis (geo, time series, streams) for fast queries and OrbitDB over IPFS for decentralized persistence.
  • Real-Time Data Streaming – Instant device-to-cloud data flow.
  • Historical Data Storage – Time series storage with query APIs.
  • Anomaly Detection – AI-based outlier detection in sensor data.
  • Edge and Cloud Rule Processing – Rules can execute locally or in the cloud depending on latency and reliability needs.

3. Automation & Workflows

  • Low-Code/No-Code Automation – Drag-and-drop workflow builder like n8n.
  • Event-Driven Triggers – Execute actions when sensor values meet conditions.
  • Third-Party Webhook Integrations – Securely send data to external services (encrypted for device to decrypt).
  • Multi-Model AI Support – Run multiple AI models (e.g., prediction, classification) on the same dataset.

4. Dashboard & Visualization

  • Dynamic IoT Widgets – Auto-generated sensor/actuator widgets from device modules.
  • Customizable Dashboards – Add, resize, and configure widgets in real time.
  • Geo-Tracking – Map view for GPS-enabled devices using Redis GEO.
  • Charts & Graphs – Real-time and historical data visualization.

5. Edge Intelligence

  • MicroPython SDK – Abstract sensor/actuator modules with dynamic command execution.
  • Multiple Sensor Types Support – Select from multiple sensor brands/models for the same operation.
  • Local AI Inference – Run AI models directly on edge devices (with RedisAI or emlearn).
  • Offline Operation – Local caching and sync when back online.

6. Security

  • End-to-End Encryption – Device ↔ Cloud data encryption.
  • Per-Device Keys – Prevents data leakage between devices.
  • Signed Firmware – Prevents malicious firmware injection.

7. Developer Tools

  • REST & WebSocket APIs – For data ingestion and retrieval.
  • SDKs – MicroPython, Arduino C, and planned Rust SDK.

8. Monetization & Ecosystem

  • Incentivized Node Network – Users can run nodes and earn rewards.
  • Marketplace for Integrations – Publish and consume automation templates and device drivers.
